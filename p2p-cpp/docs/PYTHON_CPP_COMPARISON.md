## 代码行数差异原因

### 1. 语言特性（约 2-3x 差异）
- **Python**: 动态类型、简洁语法、内置功能丰富
- **C++**: 静态类型、需要显式声明、更多样板代码

**示例对比**:
```python
# Python: 1 行
messages = [msg for msg in data if msg.type == "connect"]

# C++: 5-7 行
std::vector<Message> messages;
for (const auto& msg : data) {
    if (msg.type() == MessageType::Connect) {
        messages.push_back(msg);
    }
}
```

### 2. 测试代码（约 26x 差异）
- **Python**: 15,686 行测试
- **C++**: 587 行测试
- **原因**: C++ 测试刚开始，Python 测试已完善

### 3. DID 服务（2,081 行）
- **Python**: 完整实现
- **C++**: 未实现（不在初始需求中）

### 4. 工具和脚本（约 33,000 行）
- **Python**: 大量开发工具、构建脚本、文档生成
- **C++**: 使用 CMake，工具较少

### 5. 代码密度
- **Python**: 更多注释、文档字符串、类型提示
- **C++**: 更紧凑，头文件分离
