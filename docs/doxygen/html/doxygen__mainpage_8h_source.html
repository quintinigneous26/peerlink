4 * @section intro 项目介绍
5 *
6 * P2P Platform 是一个基于WebRTC技术的去中心化P2P通信平台，提供完整的P2P连接解决方案。
7 * 平台包含STUN服务器、TURN中继服务器、信令服务器、DID身份服务和客户端SDK等核心组件。
8 *
9 * @section features 主要特性
10 *
11 * - **智能NAT穿透**: 支持多种NAT类型的穿透，包括对称型NAT的端口预测
12 * - **运营商差异化**: 针对不同运营商的网络特性进行优化
13 * - **自动降级**: 连接失败时自动降级到中继服务
14 * - **心跳保活**: 保持连接活跃，防止NAT映射超时
15 * - **事件驱动**: 基于事件总线的异步架构
16 * - **完整的SDK**: 提供易用的客户端SDK
17 *

125 * @subsection example_basic 基本连接示例
126 *
127 * ```python
128 * import asyncio
129 * from p2p_engine import P2PEngine, P2PConfig, PeerInfo
130 *
131 * async def main():
132 * # 创建引擎
133 * engine = P2PEngine()
134 * await engine.initialize()
135 *
136 * # 创建对端信息
137 * peer_info = PeerInfo(
138 * peer_id="peer-123",
139 * public_ip="203.0.113.1",
140 * public_port=5000
141 * )
142 *
143 * # 连接
144 * result = await engine.connect_to_peer("peer-123", peer_info)
145 *
146 * if result.success:
147 * print(f"连接成功!")
148 * print(f"连接类型: {result.connection_type}")
149 * print(f"延迟: {result.latency_ms}ms")
150 * else:
151 * print(f"连接失败: {result.error}")
152 *
153 * asyncio.run(main())
154 * ```

156 * @subsection example_events 事件处理示例
157 *
158 * ```python
159 * from p2p_engine.event import EventBus, EventTopic, P2PEventType
160 *
161 * async def on_peer_connected(event):
162 * print(f"对端已连接: {event.data['peer_id']}")
163 *
164 * async def on_error(event):
165 * print(f"错误: {event.data['error']}")
166 *
167 * bus = EventBus()

169 * bus.subscribe(EventTopic.ERROR, on_error)

193 * @subsection nat_issues NAT穿透问题
194 *
195 * 如果NAT穿透失败，请检查：
196 * - STUN服务器是否可达
197 * - 防火墙是否阻止了UDP/TCP连接
198 * - 对端是否在线
199 * - 网络延迟是否过高
200 *
201 * @subsection connection_issues 连接问题
202 *
203 * 如果连接建立失败，请检查：
204 * - 信令服务器是否正常运行
205 * - 对端信息是否正确
206 * - 网络环境是否支持P2P连接
207 *
208 * @section performance 性能优化
209 *
210 * - 使用UDP而不是TCP以降低延迟
211 * - 启用多路复用以提高吞吐量
212 * - 配置合适的心跳间隔
213 * - 使用DHT加速对端发现
214 *
215 * @section security 安全性
216 *
